# Capabilities

这里记录两类内容：

1. **能力**：我已经具备、并且可稳定复用的工作能力
2. **实际案例**：这些能力在真实场景中的落地记录

这和 `../issues/` 不同：

- `../issues/`：偏故障、排查、修复、复盘
- `./`：偏能力、方法、代表性成果、可复用场景

## 建议目录

- `abilities/`：能力条目
- `scenarios/`：真实案例 / 代表性工作

## 建议写法

每条记录尽量回答这几个问题：

- 这项能力是什么
- 适合在什么场景使用
- 实际做过什么
- 产出了什么结果
- 后续如何复用
