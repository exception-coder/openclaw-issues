# 通过 skill 驱动 OpenClaw 自维护与知识沉淀

## 能力说明

我现在已经形成一套更完整的可复用能力：

> 能借助专用 skill，把 OpenClaw 自身问题处理从“临时排障”升级为“排障、沉淀、归档、推送”一体化流程。

这不只是把问题记下来，而是把一次真实处理过程抽象成可持续复用的运维能力。

## 这项能力包含什么

### 1. 识别 OpenClaw 自身问题

能够判断问题是否属于 OpenClaw 自维护范畴，例如：

- gateway / auth / token
- Control UI 状态异常
- node host / node service / system.run
- sessions / channel 显示或收发异常
- 修复后需要补文档、补索引、补模式总结

### 2. 用 skill 驱动标准化处理

当命中 OpenClaw 自维护场景时，可以通过 skill 把流程固化为标准动作：

1. 诊断与修复
2. 记录 issues
3. 更新索引与模式文档
4. 抽象为 capability / scenario
5. 提交并推送独立知识库仓库

### 3. 从“问题记录”升级为“能力记录”

现在不仅能记录某次故障，还能进一步总结：

- 我具备了什么新能力
- 这项能力适用什么场景
- 这次实战验证了什么
- 后续如何复用

## 这次实战形成的成果

- `docs/openclaw-knowledge/issues`

说明这套能力已经覆盖：

- 目录重构

- issues 与 capabilities 的关系建模
- 对实战过程做二次抽象
- 保持独立 git 仓库继续可推送

## 适用场景

适合在以下场景启用：

- 刚修完一个 OpenClaw 自身问题，想沉淀下来
- 想把一次排障整理成长期知识库
- 想把“我做过”升级成“我会做”
- 想维护一个既包含故障记录、又包含能力资产的仓库

## 复用价值

这项能力的价值在于：

- 避免知识只停留在聊天记录里
- 把一次性修复变成长期可复用资产
- 让 issues 和 capabilities 形成联动
- 让后续 OpenClaw 自维护越来越快、越来越稳
